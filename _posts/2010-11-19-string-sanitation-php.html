Personally I used a complex function that escaped everything that I thought would might cause problems. However this is not a very professional approach. Recently I found a debate on a forum regarding this aspect and one guy pointed out that for every situation:
  • mysql calls 
    mysql_real_escape_string()
  • html output 
    htmlspecialchars()
  • regular expressions 
    preg_quote()
  • use in an external command 
    escapeshellarg()/escapeshellcmd()
a standard function should suffice. The point is that you need to sanitize your string accordingly to your requirements. And probably in most of the cases you can use a build in function. Also found this :)
